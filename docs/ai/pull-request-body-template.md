# Pull Request Body Template

Use this template as a starting point. Remove empty sections instead of
leaving placeholders in the final pull request.

## Summary

- Explain the user-visible or developer-visible change.
- Mention the main files or modules touched when that adds clarity.

## Testing

- List automated tests, format checks, or manual verification.
- Say `Not run` when no verification was performed.

## Notes

- Call out risk, migration impact, follow-up work, or reviewer context.
